## 00 架构总览

**验证点**：系统采用以 US 控制平面为中心、HK 主节点和 JP 备用节点分布两侧的中心辐射拓扑。

![架构总览](<./00_分布式金融行情主备架构总览.png>)

## 01 控制平面健康状态

**验证点**：Airflow 与 Postgres 已在 US 节点启动，控制平面处于可用状态。

![控制平面健康状态](<./01_Airflow 与数据库服务健康状态.png>)

## 02-03 正常路径验证

**验证点**：正常情况下由 HK 主节点执行抓取，业务表中的来源标签为 `HK-Primary`。

![正常路径 DAG](<./02_Airflow主备切换编排图(正常路径).png>)

![HK 主节点写入结果](<./03_数据落库与溯源证明_HK主节点写入结果.png>)

## 04-06 主备切换与审计记录

**验证点**：当 HK 失败时，Airflow 激活 JP 备用节点；审计日志与业务表数据都能反映 `JP-Backup` 的写入结果。

![HK 失败后 JP 接管](<./04_故障演练_HK失败后JP自动接管.png>)

![JP 接管后的审计日志](<./05_故障审计_JP接管后的日志.png>)

![JP 接管后的数据记录](<./06_故障审计_JP接管后的数据.png>)
